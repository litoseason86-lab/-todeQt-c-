# 设置 UI 本地部署完善设计

日期：2026-07-14
状态：B 方案（左侧分类）已通过 Visual Companion 审核；目标明确为 macOS 本地部署，不做公开上线。

## 1. 背景与目标

当前 `SettingsDialog.qml` 已能管理背景主题、提示音、减少动效、计时字体、逻辑日起点、昵称，以及每日例行、科目管理和数据导出入口，但仍存在四类结构性问题：

1. 所有内容堆在一个长滚动页中，文件已超过 700 行，新增设置会继续拉长页面并放大维护成本；
2. 主题和管理入口主要依赖 `MouseArea`、`TapHandler`，键盘焦点、读屏语义和可见焦点不完整；
3. 小字号弱化文字和部分操作目标偏小，浅色、深色背景下的可读性不稳定；
4. 多数 `AppSettings` setter 在 `QSettings::sync()` 后不检查写入状态，界面可能把失败误报成成功。

本次目标是把设置弹窗完善为适合这台 Mac 长期本地使用的设置中心：结构清晰、状态可靠、键盘可操作、在项目最小窗口中不裁切，并保持现有暖纸/玻璃设计语言。

## 2. 范围边界

### 2.1 本次包含

- 设置弹窗改为“左侧分类 + 右侧当前页”的桌面双栏结构；
- 外观、专注、通用、数据与管理、关于五个分类；
- 将工作时长和休息时长集中到设置，同时保留专注页的情境快捷入口，两处共用同一份 `AppSettings` 状态；
- 修正主题默认值和旧主题 ID 在设置画廊中的选中状态；
- 设置写入成功、失败与回滚反馈；
- 完整键盘导航、可见焦点、读屏名称、减少动效路径；
- 页面懒加载、主题缩略图解码限制和实时效果层数量约束；
- 离屏测试、完整构建及 `/Applications/番茄Todo.app` 本地部署验证。

### 2.2 本次不包含

- App Store 上架、公证、发布渠道或自动更新器；
- 云同步、账号、遥测或任何远程服务；
- 多语言和 RTL 布局；
- 通知、导入、音量、恢复默认值等当前没有完整后端语义的占位设置；
- 与设置页无关的全应用视觉重构。

本地部署只是缩小分发范围，不降低数据可靠性、可读性和窗口适配要求。

## 3. 已确认的设计决策

Visual Companion 依次审核并确认：

1. 采用 **B 方案：左侧分类**，不采用继续增长的单页长列表，也不采用容量有限的顶部五标签；
2. 外观页采用响应式六主题画廊，内容区不叠加多层玻璃卡片；
3. 专注、通用、数据与管理、关于四页的内容边界通过；
4. 交付目标改为 macOS 本地部署，不按公开上线范围扩张。

本文档取代 `2026-07-07-settings-hub-sidebar-declutter-design.md` 中以下旧决策：

- “番茄/休息时长只留在专注待机页”改为“设置页集中管理，专注页保留同源快捷入口”；
- “弹窗进出和 Switch 圆钮动画不受减少动效控制”改为“所有非必要设置页动画均受减少动效控制”。

旧文档中侧栏减负、管理入口迁移和全局减少动效机制继续有效。

## 4. 信息架构

### 4.1 外观

- 背景主题：暖色、粉色、烟雨江南、星空、雨夜窗景、月夜山影；
- 减少动效；
- 纤细计时字体。

背景画廊只在外观页加载。每个候选项使用自身主题的缩略图和明暗模式预览，选中态同时使用边框、勾选标记和读屏状态，不能只靠颜色。

### 4.2 专注

- 每轮专注：5–180 分钟；
- 短休息：1–60 分钟；
- 阶段完成提示音。

设置页和 `FocusView` 都读写 `AppSettings.workMinutes`、`breakMinutes`、`soundEnabled`。专注页的快捷设置不形成第二份状态，也不维护独立默认值。

### 4.3 通用

- 昵称：最多 12 个字符，保存前去除首尾空白；
- 每日起始时间：00:00–06:00；
- 起始时间下方显示人类可读的日期口径，例如“00:00–03:59 仍计入前一天”。

昵称不按每个按键写盘。回车、失焦或切换分类时提交；写入失败时保留草稿、显示错误，并阻止销毁包含未保存草稿的页面。

### 4.4 数据与管理

- 每日例行；
- 科目管理；
- 数据导出；
- 本机数据说明：“任务、专注记录和偏好默认保存在这台 Mac，不会自动上传。”

三个管理入口继续打开现有独立弹窗，不在设置页复制管理功能。入口改为可聚焦的 Qt Quick Controls `Button` 或 `ItemDelegate`，不再使用只有指针语义的透明矩形。

### 4.5 关于

- 应用名“番茄 Todo”；
- 版本号读取 `Qt.application.version`，不在 QML 中硬编码；`main.cpp` 使用 CMake 的 `PROJECT_VERSION` 设置应用版本；
- 简短说明本应用为本地任务与专注工具；
- 明确当前没有云同步和远程上传。

关于页不添加官网、检查更新、反馈等无实现入口。

## 5. 弹窗结构与响应式布局

### 5.1 桌面布局

- 弹窗宽度上限 760 px，实际宽度不超过父窗口左右各 32 px 的安全区；
- 高度上限 640 px，实际高度不超过父窗口上下各 32 px 的安全区；
- 左侧分类栏宽 168 px；
- 右侧由固定页头、当前页滚动内容、固定状态栏组成；
- 关闭按钮放在右侧页头，`Escape` 同样关闭；
- 关闭后恢复到打开设置前的控件焦点。

项目当前最小窗口为 860×620。该尺寸下弹窗宽度保持 760 px，高度收缩到 556 px，只有右侧当前页内容区滚动；分类栏、页头和状态栏不随内容滚走。

### 5.2 窄宽降级

当弹窗可用宽度低于 640 px 时：

- 左侧分类改为顶部两列分类按钮；
- 主题画廊从三列改为两列；
- 设置行由“左说明 + 右控件”改为上下单列；
- 所有说明文字允许换行，不出现横向滚动。

该分支主要用于组件测试和未来窗口约束变化，不改变当前 860 px 的应用最小宽度。

## 6. 视觉与交互规范

### 6.1 玻璃范围

- 设置弹窗整体是重要浮层，可使用 `Theme.glassDialog`、边缘高光和一层 `MultiEffect` 阴影；
- 分类栏允许使用同一弹窗内部的半透明色区分，但不新增第二个实时模糊采样；
- 右侧内容区、设置行和分隔线保持清晰，不把每一行或每一组再做成玻璃卡片；
- `Theme.glassBlurAllowed === false` 时使用现有高不透明度纯色降级；
- 整个设置弹窗最多一个 `layer.enabled` 效果层，主题缩略图不启用离屏特效。

### 6.2 文字与操作目标

- 设置正文基准字号调整到 15–16 px；
- 次要说明不得低于 13 px，并使用能在浅色和深色表面达到 4.5:1 的文字令牌；
- 标题、正文、说明最多使用三档字号；
- 按钮、Switch 命中区、步进按钮和管理入口最小为 44×44 px；
- 自绘控件通过外部 `Control` 提供命中区、焦点和可访问性语义，视觉轨道本身可以小于 44 px。

本次优先在设置组件内引入适合桌面阅读的字号角色，不直接放大全应用全部字号，避免无关页面回归。

### 6.3 键盘与读屏

- Tab 顺序为分类栏从上到下，再进入当前页从上到下，最后到关闭按钮；
- 分类按钮、主题选择、Switch、时长步进器和管理入口都支持 `Space`/`Enter`；
- 每个交互控件使用 `activeFocus` 显示明确焦点环；
- 主题项提供 `Accessible.name` 和选中状态；
- 管理入口的读屏名称包含动作，例如“打开每日例行管理”；
- 弹窗内不允许焦点陷阱，`Escape` 始终可关闭。

### 6.4 动效

- 弹窗进入 200–220 ms，退出 160–200 ms，只动画 `opacity` 和 `scale`；
- 分类切换最多使用 120–160 ms 的透明度过渡，不动画宽高和 anchors；
- Switch 圆钮动画控制在 120 ms；
- `reduceMotion` 开启时，上述非必要动画时长统一为 0，状态直接切换；
- 不添加循环动画。

## 7. 组件拆分

`SettingsDialog.qml` 只保留弹窗壳、分类状态、页面 Loader、关闭和状态反馈。新增设置子组件目录：

```text
qml/components/settings/
  SettingsNavigation.qml
  SettingsAppearancePage.qml
  SettingsFocusPage.qml
  SettingsGeneralPage.qml
  SettingsDataPage.qml
  SettingsAboutPage.qml
  SettingsSection.qml
  SettingsRow.qml
  SettingsSwitch.qml
  SettingsThemeChoice.qml
```

职责边界：

- `SettingsNavigation`：分类选择、窄宽重排和焦点顺序；
- 五个页面：只组合本分类设置并调用 `appSettingsRef`；
- `SettingsSection`：标题、说明和分隔结构，不承担持久化；
- `SettingsRow`：统一标签、说明、控件槽位、最小高度和可访问性布局；
- `SettingsSwitch`：统一开关命中区、视觉轨道、键盘语义和减少动效；
- `SettingsThemeChoice`：封装单个主题缩略图、选中标记、焦点环和读屏状态；
- `SettingsDialog`：监听页面切换、提交待保存草稿、显示成功或失败状态，并转发现有管理信号。

当前页使用一个 `Loader`。切换分类前，壳调用当前页可选的 `commitPendingEdits()`；返回失败时留在当前页并聚焦错误控件。这样既限制内存和图片解码，也不会销毁未保存昵称。

## 8. 数据流与持久化

### 8.1 AppSettings 归一化

- `backgroundTheme` 新安装默认值从旧 ID `warmPaper` 改为 `warm`；
- `SettingsAppearancePage` 用 `Theme.migrateThemeId()` 比较选中状态；
- `MainWindow` 现有旧 ID 写回逻辑保留，确保持久化值最终为新 ID；
- `workMinutes` 在读取和写入时归一化到 5–180；
- `breakMinutes` 在读取和写入时归一化到 1–60；
- 非法或损坏的时长配置回退到默认值 25 和 5，不简单 clamp 到边界，避免把损坏值误解释成用户选择。

### 8.2 写入失败

`AppSettings` 增加统一私有写入辅助函数：

1. 保存旧值和“键是否原本存在”；
2. 写入新值并调用 `sync()`；
3. 检查 `QSettings::status()`；
4. 成功时才发对应 `...Changed` 信号；
5. 失败时恢复旧值或删除新建键，再次同步，并发出 `settingsWriteFailed(key, message)`；
6. 不发成功变更信号。

`SettingsDialog` 监听失败信号，在固定状态栏显示“无法保存设置，请检查系统权限后重试”。失败状态不使用自动消失 Toast，避免用户错过。下一次成功写入后，状态栏恢复为“所有设置已保存到本机”。

## 9. 测试策略

### 9.1 C++

- 背景主题新默认值为 `warm`，旧值仍能由 QML 迁移；
- `workMinutes`、`breakMinutes` 默认值、合法值、损坏值和边界；
- 每个设置同值不重复发信号；
- 正常写入后重建 `AppSettings` 仍可读取；
- 使用不可写的临时设置路径复现写入失败，验证旧值恢复、失败信号发出、成功信号不发出。

### 9.2 QML

- 分类栏五项存在，点击和键盘激活都能切换页面；
- 只有外观页激活时创建六张主题缩略图；
- 旧主题 ID 在画廊中显示正确选中项；
- 专注时长、休息时长、提示音与 mock `AppSettings` 双向同步；
- 昵称提交成功、失败保留草稿、失败时阻止页面销毁；
- 三个管理入口发出既有信号；
- 关于页版本来自 `Qt.application.version`；
- `reduceMotion` 开启时弹窗、页面切换和 Switch 动画均不运行；
- 860×620 与窄宽测试下无横向滚动，关闭按钮和错误状态始终可达；
- 测试 mock 包含生产组件读取的全部属性，定向测试运行时零 QML 警告。

所有自动测试继续使用：

```bash
QT_QPA_PLATFORM=offscreen QT_QUICK_CONTROLS_STYLE=Basic
```

自动流程不执行 `open`，不主动拉起应用窗口。

## 10. 本地构建与部署验收

每次完整交付按以下顺序：

1. 运行设置页定向测试；
2. 运行相关 C++ 服务测试；
3. 完整构建并运行全量 CTest；
4. 由 CMake `deploy-local-app` 目标部署到 `/Applications/番茄Todo.app`；
5. 核对应用包内二进制时间戳对应本次构建；
6. 不自动启动 GUI；用户退出旧进程后手动重新打开，进行本机视觉验收。

本地验收通过条件：

- 定向测试和全量测试通过；
- 测试与离屏运行零 QML 警告；
- 860×620 无裁切、无横向滚动；
- 设置修改在应用重启后保持；
- 写入失败不会显示成功状态；
- 六张主题缩略图仅在外观页按目标尺寸解码；
- 设置弹窗没有新增多重实时模糊或逐行离屏层；
- 最新应用包已部署到 `/Applications/番茄Todo.app`。

## 11. 实施顺序约束

后续实施计划必须遵守项目分阶段规则：

1. 先创建设置子组件文件框架，只放主要结构、属性、信号和导入；
2. 完成弹窗壳与分类导航；
3. 完成外观页；
4. 完成专注与通用页；
5. 完成数据与管理、关于页；
6. 完成持久化失败反馈、动效门控和测试；
7. 最后统一检查导入、调用关系、格式、构建和本地部署。

每一阶段只处理当前目标，发现组件职责或调用关系不清时先修正结构，再继续下一阶段。
