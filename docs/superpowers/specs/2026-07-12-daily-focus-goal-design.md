# 每日专注目标：任意小时与分钟输入设计

日期：2026-07-12  
状态：用户已通过可视化伴随确认

## 目标

在仪表盘右侧专注面板内提供“今日专注目标”。用户每天为当前逻辑日重新设置一次目标，可以分别输入任意小时和分钟；目标不继承到下一逻辑日。仪表盘持续展示今日累计专注时长、目标时长、完成百分比和达成状态。

## 已确认决策

- 输入方式为两个独立数字输入框：小时和分钟。
- 不使用固定步进，不要求用户输入 `HH:MM` 字符串或总分钟数。
- 每个逻辑日独立设置；跨逻辑日后不继承昨天目标。
- 当天未设置时显示“尚未设置”和“设置今日目标”，不计算完成百分比。
- 目标范围为 1 分钟至 24 小时。
- 目标卡使用内联编辑，不增加新的弹窗。
- 目标卡本身不增加实时模糊；它复用右侧专注面板唯一的背景采样与模糊层。
- 最终交付仅使用 Qt 6、QML、C++ 和预编译 Qt Shader。禁止生成或引入 SwiftUI、UIKit、CSS、React 实现。

## 范围

### 包含

- 今日目标的设置、修改、校验、持久化和跨逻辑日失效。
- 未设置、编辑、进行中、目标达成四种界面状态。
- 小时与分钟键盘输入、Tab 顺序、Enter 保存、Escape 取消。
- 专注面板的单层背景采样、模糊、边缘折射、高光和纯色降级。
- C++ 设置测试、QML 行为测试、Shader 资源守门和无界面全量回归。

### 不包含

- 历史每日目标查询、连续达标统计或目标趋势图。
- 多套目标模板、自动推荐、周期目标或提醒通知。
- 数据库新表。当前只保存当天目标日期和分钟数。
- 在普通内容卡、统计卡或任务卡上增加实时模糊。

## 逻辑日与存储

### 设置项

`AppSettings` 不再暴露旧的 `dailyFocusGoalHours`。改为以两个 QSettings 值保存当前记录：

- `focus/dailyGoalDate`：ISO 日期 `yyyy-MM-dd`，表示设置目标时对应的逻辑日。
- `focus/dailyGoalMinutes`：目标总分钟数，合法范围 `1..1440`。

旧的 `focus/dailyGoalHours` 尚未正式发布，不迁移为今天的目标。自动迁移会把一个没有日期语义的永久设置错误地套用到当前日，违背“每天重新设置”。

### C++ 接口

`AppSettings` 提供两个 QML 可调用接口：

- `dailyFocusGoalMinutesForDate(const QString &isoDate)`：仅当存储日期与参数相等时返回分钟数，否则返回 `0`，表示未设置。
- `setDailyFocusGoal(const QString &isoDate, int minutes)`：验证 ISO 日期和分钟范围后一次性写入日期与分钟，`sync()` 后发出一次 `dailyFocusGoalChanged`；失败时不修改旧值并返回 `false`。

同一日期、同一分钟的重复保存不写盘、不重复发信号。C++ 是最终数据边界，不能只依赖 QML 校验。

### 当前逻辑日

`DashboardView` 使用现有 `LogicalDay.js` 和 `settingsRef.dayStartHour` 计算当前逻辑日 ISO 日期。`logicalDayService.changed` 触发时重新计算日期并重载目标：

- 新逻辑日没有目标时立即进入“未设置”状态。
- 修改每日起始时间导致逻辑日变化时使用同一条失效路径。
- 当前日期不直接由物理午夜决定，保持与任务和统计服务一致。

## 界面设计

目标平台沿用现有项目：macOS 桌面应用、矩形窗口、默认 `1024×768`、最小 `860×620`、Qt Quick Controls Basic、中文、键盘和鼠标输入。

### 未设置

- 标题：“今日专注目标”。
- 主状态：“尚未设置”。
- 辅助说明：“为今天定一个清晰、可完成的目标。”
- 主操作：“设置今日目标”。
- 不显示百分比和伪造的 `0%` 进度。

### 编辑

- 标题：“设置今日专注目标”。
- 两个 `TextField`：小时、分钟。
- 小时允许 `0..24`，分钟允许 `0..59`。
- `0小时0分钟` 无效。
- `24小时` 时分钟必须为 `0`。
- 校验错误显示在输入区下方，使用文字说明，不只改变边框颜色。
- “保存”为主操作，“取消”为次操作。
- Enter 保存，Escape 取消；Tab 顺序为小时、分钟、取消、保存。
- 保存失败保留编辑内容并显示“目标保存失败，请重试”。

### 已设置/进行中

- 显示今日累计专注时长，格式为 `HH:MM:SS`。
- 显示目标时长，格式为 `目标 HH:MM`。
- 显示完成百分比，向下取整并限制在 `0..100`。
- 提供“修改目标”入口，重新进入内联编辑。
- 进度条只承担辅助视觉，百分比文字是主信息。

### 目标达成

- 累计时长继续增长，不能截断为目标值。
- 百分比显示上限仍为 `100%`。
- 同时显示勾选符号和“目标达成”，不只依赖绿色。
- 用户仍可修改当日目标。

## QML 组件边界

### `FocusGoalCard.qml`

- 接收 `totalSeconds`、`goalMinutes`、`hasGoal`。
- 管理 `unset`、`display`、`editing` 三个离散状态。
- 派生累计时钟、目标时钟、百分比和达成状态。
- 发出 `goalSubmitted(int totalMinutes)`，不读写 `AppSettings`。
- 通过 `Loader` 按需创建编辑器；非编辑状态销毁编辑器对象。

### `DailyFocusGoalEditor.qml`

- 负责两个输入框、整数校验、组合边界校验、键盘导航和错误文案。
- 接收当前目标分钟数并拆分为小时与分钟。
- 只发出 `submitted(int totalMinutes)` 和 `cancelled()`。
- 不读取时间、统计服务或设置服务。

### `DashboardTimerPanel.qml`

- 接收由 `DashboardView` 注入的目标分钟数和当前逻辑日。
- 组合计时器、目标卡和外层光学背景。
- 把目标提交信号向上转发，不直接存储日期目标。
- 实时累计继续使用“已落库今日秒数 + 当前有效工作会话秒数”，休息阶段不累计。

### `DashboardView.qml`

- 作为页面协调层计算当前逻辑日、读取当天目标并响应设置变化。
- 收到目标提交后调用 `AppSettings.setDailyFocusGoal(logicalDate, minutes)`。
- 保存成功后由 `dailyFocusGoalChanged` 驱动重载；保存失败设置页面错误状态。

## Liquid Glass 光学层

实时玻璃只保留在右侧专注面板，因为它同时承担计时和目标设置，属于重要控制组件。目标卡、任务卡和统计卡保持清晰的半透明内容层，不各自做背景采样。

### 四层结构

1. **背景采样与模糊**：`ShaderEffectSource` 截取专注面板对应的静态壁纸区域，单个 `MultiEffect` 执行有限半径模糊。
2. **边缘折射**：单个 `ShaderEffect` 使用圆角矩形 SDF，在边缘约 6～8px 范围内向中心偏移采样；中心区域保持中性，不扭曲文字背后的主体背景。
3. **半透明着色**：使用现有 `Theme.glassCard`/深色对应令牌控制文字对比度。
4. **受光与内容**：顶部高光、底部弱暗边和 1px 玻璃描边位于业务内容下方；业务内容始终在最高层。

### Shader 约束

- Qt 6 `ShaderEffect` 引用由构建系统预编译的 `.qsb` 片元着色器。
- 着色器采用凸圆角矩形 SDF，边缘采样方向指向中心，避免凹面越界采样伪影。
- 不做 RGB 三通道色散；三次采样带来的成本不符合本项目 60 FPS 优先级。
- 折射强度保持克制，只用于边缘厚度感，不能影响计时数字和输入框可读性。
- Shader 尺寸、圆角和边缘宽度由 QML 属性传入，窗口缩放后保持几何一致。

### 采样生命周期与性能

- `ShaderEffectSource.live` 设为 `false`；仅在壁纸主题、面板几何或采样矩形变化时调用 `scheduleUpdate()`。
- 光学子树通过 `Loader.active` 受 `Theme.glassBlurAllowed`、壁纸引用和面板可见性共同控制。
- 禁用模糊或面板不可见时销毁 `ShaderEffectSource`、`MultiEffect` 和 `ShaderEffect`。
- 只对光学效果子树使用 `layer.enabled`，不缓存整个计时业务面板。
- 动效不改变 Shader 参数，不逐帧重算折射。

### 降级

`Theme.glassBlurAllowed == false`、壁纸引用缺失或 Shader 不可用时：

- 不创建背景采样、模糊和折射对象。
- 外层改用 `Theme.glassSolidCard`/`glassSolidHover` 等高不透明度纯色令牌。
- 玻璃描边和顶部高光可以保留，但不能依赖它们保证文字对比度。
- 浅色与深色主题均使用 `inkStrong`、`inkSoft`、`accentInk` 等语义令牌，不写死字色。

## 动效与无障碍

- 编辑状态切换只动画 `opacity` 或 `scale`，使用 `Behavior` 与 `NumberAnimation`，时长 `180..220ms`。
- 不动画宽高、anchors 或复杂布局几何。
- `reduceMotion` 开启时立即切换状态，不播放过渡。
- 所有操作使用 Qt Quick Controls，支持键盘焦点和屏幕阅读器名称。
- 可见焦点边框不能只依赖 hover。
- 文本与背景目标对比度不低于 4.5:1；控件边界不低于 3:1。
- 达成和错误状态均以文字配合形状表达，颜色不是唯一信息来源。

## 错误处理

- QML 输入非法：不调用 C++，原位显示明确错误。
- C++ 拒绝日期或分钟：保留编辑状态并显示保存失败，不覆盖已有合法目标。
- 设置读取到损坏分钟值或日期不匹配：按未设置处理。
- 光学效果初始化失败：退回纯色玻璃，不阻断目标设置和计时功能。

## 测试设计

### C++ `ServiceTests`

- 同一逻辑日期保存并读取任意分钟目标。
- 重建 `AppSettings` 后目标仍可读取。
- 查询不同日期返回 `0`。
- 拒绝空日期、非法 ISO 日期、`0`、负数和大于 `1440` 的分钟数。
- 非法保存不覆盖已有合法值。
- 同值保存不重复发信号。
- 新日期保存替换当前记录并只发一次信号。

### QML `tst_dashboard_view.qml`

- 未设置状态不显示百分比，提供设置入口。
- 当前分钟数正确拆为小时和分钟。
- `0:00`、`24:01`、小时或分钟越界显示错误。
- `0:01`、`2:20`、`24:00` 可以提交正确总分钟数。
- 取消编辑不发保存信号。
- Enter、Escape 和 Tab 顺序可用。
- 保存失败保留编辑状态和输入值。
- 目标进度向下取整并封顶 `100%`。
- 达成状态同时显示勾选、文字和完整进度。
- `logicalDayService.changed` 后日期不匹配立即回到未设置状态。
- 进行中工作会话计入实时秒数，休息阶段不计入。

### 光学与资源守门

- `Theme.glassBlurAllowed=false` 时光学 `Loader` 不活跃并使用纯色令牌。
- 启用时页面最多创建一套背景采样、模糊和折射对象。
- `.qsb` Shader 资源已打包并可从 QRC URL 解析。
- 浅色和深色主题下目标卡使用语义字色与玻璃令牌。

### 验证命令

所有自动验证使用：

```bash
QT_QPA_PLATFORM=offscreen QT_QUICK_CONTROLS_STYLE=Basic ctest --test-dir <临时构建目录> --output-on-failure
```

构建目录放在仓库外，自动流程不打开 `.app` 或任何 GUI 窗口。

## 验收标准

- 用户可以直接输入并保存 `2小时20分钟` 等任意合法目标。
- 第二逻辑日不继承前一日目标，未设置状态明确。
- 目标进度随已落库和进行中有效专注实时更新。
- 浅色、深色壁纸下文字和输入边界清晰可读。
- 页面只有右侧专注面板一套实时玻璃效果。
- 禁用模糊后业务功能、布局和可读性不下降。
- 全量构建、C++ 测试、QML 测试和资源守门全部通过。
