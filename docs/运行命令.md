# 运行命令（VSCode 终端）

在项目根目录 `/Users/zerionlito/code/番茄todo` 下执行。Qt 安装路径若有变动，把 `CMAKE_PREFIX_PATH` 换成对应目录。

## Debug 构建（日常开发）

构建完成后会自动把最新应用包同步到固定启动入口 `/Applications/番茄Todo.app`。

```bash
cmake -B build -S . -DCMAKE_PREFIX_PATH=/Users/zerionlito/Qt/6.9.0/macos
cmake --build build -j8
```

可选开关：

```bash
# 跳过自动部署（审计/CI/临时目录构建用，避免覆盖固定入口）
cmake -B build-audit -S . -DCMAKE_PREFIX_PATH=/Users/zerionlito/Qt/6.9.0/macos \
      -DPOMODORO_TODO_DEPLOY_LOCAL=OFF

# 关闭 QML 调试钩子（非调试期建议关闭，部署包更干净）
cmake -B build -S . -DCMAKE_PREFIX_PATH=/Users/zerionlito/Qt/6.9.0/macos \
      -DPOMODORO_TODO_ENABLE_QML_DEBUG=OFF
```

## Release 构建

```bash
cmake -B build-release -S . -DCMAKE_PREFIX_PATH=/Users/zerionlito/Qt/6.9.0/macos \
      -DCMAKE_BUILD_TYPE=Release -DPOMODORO_TODO_DEPLOY_LOCAL=OFF
cmake --build build-release -j8
```

产物：`build-release/PomodoroTodo.app`。

## 测试

```bash
# 全套（8 个目标：C++ 180 用例 + QML 29 个测试文件）
ctest --test-dir build --output-on-failure

# 只跑单个测试目标
./build/PomodoroTodoTests        # 服务层主套件
./build/CoreLogicTests           # 核心逻辑边界补充
./build/RobustnessTests          # 健壮性与性能
./build/CountdownServiceTests    # 倒计时服务

# 只跑 QML 测试
ctest --test-dir build -R PomodoroTodoQmlTests --output-on-failure

# 按名字过滤 C++ 用例（Qt Test 原生参数）
./build/PomodoroTodoTests interruptedFocusRestoresPausedAndKeepsProgress
```

## 启动

```bash
# 日常入口（Debug 构建自动同步到这里）
open /Applications/番茄Todo.app

# 直接跑构建产物（调试时看终端日志）
./build/PomodoroTodo.app/Contents/MacOS/PomodoroTodo
```

应用有单实例守卫：已有实例在运行时，第二个实例打印「番茄Todo 已在运行」后退出。

## 常见问题

- **改了 QML 但界面像旧版**：先清 qmlcache 再启动：
  `rm -rf ~/Library/Caches/PomodoroTodo/PomodoroTodo/qmlcache`
- **可选 QML 静态检查**：
  `/Users/zerionlito/Qt/6.9.0/macos/bin/qmllint -I qml qml/*.qml qml/views/*.qml qml/components/*.qml`
